Formulário

Formulário Simples

Formulários são elementos fundamentais para coletar dados de usuários. A estrutura básica de um formulário começa com a tag <form>, que serve como contêiner para todos os campos de entrada, botões e instruções.

Exemplo de formulário:

Dados simples

  • <form> - Define o formulário. Aceita atributos como:
    • action - URL para onde os dados serão enviados ao clicar em “Enviar”, ou seja, o backend.
    • method - Método de envio GET ou POST.
  • <lable> - Rótulo de um campo, deve ser associado ao id do input correspondente via for.
  • <input> - Campo de entrada de dados, pode variar conforme o tipo (text, email, number, etc.).
  • <input type="submit"> - Botão para enviar o formulário, onde o escrito no botão é correspondente do atributo value.
  • <fieldset> - Agrupam campos relacionados como uma seção, uma borda em volta do formulário.
  • <legend> - Define a legenda do formulário, nesse caso as palavras "Dados Simples".

Associando Lables aos Inputs

A associação entre lables e inputs melhora a usabilidade e a acessibilidade. Quando o usuário clica na lable, o input correspondente é automaticamente selecionado, o que ajuda na experiência do usuário.

Para fazer isso é muito simples, o for da <lable> deve ser igual ao id do <input>, como na imagem acima. Dessa maneira os dois se associam e essa ligação também facilita a leitura por leitores de tela.

Método GET e POST

Ao enviar um formulário, os dados podem ser transmitidos de duas formas: via GET ou POST, definidos no atributo method do formulário.

GET

É o método padrão, usado se o method não for especificado. É visível na URL. É utilizado para campos simples como cadastros, peso, altura, buscas e informações que não sejam sensíveis, ideal para buscas, filtros ou dados não confidenciais. Get pode ser resumido como método para obter.

O limite para uso do GET são de 3 mil bites, aproximadamente 3 mil caracteres.

POST

Para usar deve alterar o atributo method. Não exibe os dados na URL. Recomendado para informações sensíveis como senhas, endereços, dados bancários, envio de arquivos, etc. Sem limite prático de tamanho.

Tipos de Inputs

Text

Texto simples, usados para nomes, sobrenomes e etc.

Password

Oculta os caracteres digitados. Ideal para senhas.

Number

Não aceita letras, apenas números.

Email

Valida se há "@" e formato de e-mail.

Tel

Entrada de telefone. Pode usar pattern para validar.

O atributo pattern é usado para validar o formato desejado, usando as expressões regulares. Nesse exemplo o telefone deve conter dois dígitos numéricos entre parenteses, depois quatro ou cinco dígitos, um hífen e mais quatro dígitos.

O atributo placeholder é usado para mostrar no campo do input o formato desejado.

Month

Para mês e ano.

Date

Para datas completas, com dia mês e ano.

Time

Para horas e minutos.

Color

Utilizado para selecionar uma cor, dentre todas as posibilidades no formato exadecimal.

Range

Barra deslizante para valores numéricos, que pode enviar valores.
Obs: Por padrão os valores iniciais são de 0 à 100, mas podemos acrescentar os atributo min e max para limitar essa variação, já o atributo value é usado para indicar onde a barra vai iniciar, como uma sugestão. Nesse exemplo eu deixei de 0 a 6.

File

Para fazer upload de arquivos. Use sempre com method="post".

CheckBox

Permite selecionar múltiplas opções. Nesse caso o <input> vem antes da <lable>.

Esportes




Radio

Seleção única dentro de um grupo.

Sexo

É necessário que todos os inputs de um radio tenham o mesmo name, pois fazem parte do mesmo grupo, assim o usuário só pode selecionar uma opção do grupo, ao clicar em uma opção as outras automaticamente desmarcam.

O value de cada radio é o dado que será enviado se aquela opção for escolhida.

Textarea

Área de texto para mensagens longas, use com POST para evitar vazamento de dados pela URL.


Submit

Envia os dados do formulário, com um botão(input) submit.

Reset

Limpa os campos preenchidos, com um botão(input).

Select

Cria uma lista suspensa com opções pré-definidas.

Use <optgroup> para organizar grupos de opções.

Pode se usar o atributo disable para deixar uma opção desabilitada.

Datalist

Abre uma caixa com opções para selecionar, porém é permitido digitar caso algumas das opções apresentadas não seja suficiente.

Atributos Importantes para Inputs

required

Torna o campo obrigatório.

minlength

Limita o número mínimo de caracteres a ser preenchido, nesse exemplo o minlength é de 15.

maxlength

Limita o número máximo de caracteres a ser preenchido, nesse exemplo o maxlength é de 5.

size

Aumenta visualmente o comprimento do <input>.

placeholder

Texto de dica dentro do <input>.

autocomplete

Sugere dados salvos pelo navegador. Ex: username, email, new-password, current-password e etc.

min

Define valores mínimo para number, date, etc, nesse exemplo a altura miníma é 100.

max

Define valores máximo para number, date, etc, nesse exemplo a idade máxima é 18.

step

Limita o quanto um número pode variar ex: step="0.5" vai aceitar somente números quebrados com 0.5 de diferença.

value

Sugere um valor para o campo a ser preenchido.

checked

Pré-seleciona um <input> do tipo radio ou checkbox, nesse exemplo os campos basquete e jiu jitsu estão com o atributo checked.
