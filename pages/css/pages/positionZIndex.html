CSS

Position e Z-Index

As propriedades de posicionamento no CSS permitem controlar a localização exata de um elemento na página, tirando-o do fluxo normal do documento ou ajustando sua posição em relação a outros elementos. O z-index complementa o posicionamento, controlando a ordem de empilhamento dos elementos.

Position

A propriedade position é fundamental para o layout de elementos e determina como um elemento é posicionado no documento. Os valores mais comuns são:

Static (Padrão)

Este é o valor padrão para todos os elementos HTML. Elementos com position: static são posicionados de cima para baixo, tudo alinhado a esquerda. As propriedades top, right, bottom, left e z-index não têm efeito sobre elementos estáticos.

Nesse exemplo, tenho um um parágrafo, uma imagem e outro parágrafo. Eles são exibidos na ordem em que aparecem no HTML, um abaixo do outro, sem qualquer posicionamento especial.

Como usar:

.elemento-estatico {
  position: static;
}

Relative

Elementos com position: relative tem sua posição relativa ao position: static e sua posição pode ser ajustada usando as propriedades top, right, bottom, left. O espaço original que o elemento ocuparia no position: static é mantido, e outros elementos não se movem para preenchê-lo.

Nesse exemplo, a imagem é movida 30px para a direita de sua posição original, mas o espaço que ele ocuparia originalmente ainda está reservado.

Como usar:

.elemento-relativo {
  position: relative;
  left: 30px;
}

Absolute

Elementos com position: absolute leva em consideração quem é o pai do elemento. Lembrando que o navegador vai procurar o ancestral mais próximo que tenha position relative, absolute, fixed ou sticky. Se não houver um ancestral posicionado, o elemento será posicionado em relação ao <body> do documento. Elementos absolutos não ocupam espaço no layout, e outros elementos se comportam como se eles não existissem.

Se eu quiser posicionar o elemento vermelho no canto superior direito da <div>, eu defino a <div> (que é o pai) como position: relative; e o filho como position: absolute;, top: 0; e right: 0;. O resultado será:

Como usar:

.container-pai {
  position: relative;
  /* Essencial para que o filho absoluto se posicione em relação a ele */
  width: 300px;
  height: 200px;
  border: 1px solid black;
}
.elemento-absoluto {
  position: absolute;
  top: 10px;
  right: 10px;
  width: 50px;
  height: 50px;
}

Neste caso, .elemento-absoluto será posicionado 10px do topo e 10px da direita de .container-pai.

Fixed

Elementos com position: fixed são posicionados em relação à viewport (a janela visível do navegador). Eles permanecem na mesma posição mesmo quando a página é rolada. São comumente usados para cabeçalhos, rodapés ou botões de "voltar ao topo" que devem estar sempre visíveis.

Nesse exemplo, a imagem é posicionada fixamente no topo e a direita, mesmo que a página seja rolada. O elemento não se move com a rolagem da página.

Como usar:

.cabecalho-fixo {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  text-align: center;
}

Sticky

Elementos com position: sticky são posicionados de modo padrão até que atinjam um determinado limite de rolagem (definido por top, right, bottom, left), momento em que se tornam "fixos" (como position: fixed) em relação à viewport. É útil para elementos como menus de navegação que devem rolar com a página até um certo ponto e depois permanecer visíveis no topo.

Nesse exemplo, o menu está posicionado um pouco abaixo do topo, ao rolar a página, o menu rola normalmente com a página até que atinja o topo da viewport, onde então se fixa no topo, tendo como resultado:

Como usar:

.menu-sticky {
  position: sticky;
  top: 0;
  /* Fica fixo no topo quando a rolagem atinge 0px do topo */
  padding: 10px;
}

top, right, bottom, left

Essas propriedades são usadas em conjunto com position (exceto static) para especificar o deslocamento de um elemento a partir das bordas de seu contêiner de posicionamento.

top: Define a distância do elemento em relação à borda superior do contêiner.

right: Define a distância do elemento em relação à borda direita do contêiner.

bottom: Define a distância do elemento em relação à borda inferior do contêiner.

left: Define a distância do elemento em relação à borda esquerda do contêiner.

Exemplo:

.caixa-posicionada {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  /* Centraliza o elemento */
  width: 100px;
  height: 100px;
}

Z-Index

A propriedade z-index controla a ordem de empilhamento de elementos posicionados (aqueles com position diferente de static). Elementos com um z-index maior aparecem na frente de elementos com um z-index menor. Funciona como camadas, onde números maiores significam camadas mais próximas do observador.

Como usar:

Contexto de Empilhamento

O z-index só funciona dentro de um "contexto de empilhamento". Um contexto de empilhamento é formado por elementos que têm uma propriedade position diferente de static e um z-index definido, ou por outros elementos que criam um novo contexto de empilhamento (como elementos flex ou grid, ou elementos com opacity menor que 1, entre outros). Dentro de um contexto de empilhamento, os z-index dos elementos filhos são comparados. Elementos em diferentes contextos de empilhamento são empilhados com base no z-index de seus contextos de empilhamento.

Pontos Chave:

  • O z-index só afeta elementos que não têm position: static.
  • Um elemento filho não pode estar abaixo de seu elemento pai no empilhamento, mesmo que o filho tenha um z-index negativo, se o pai criar um contexto de empilhamento.
  • Entender o contexto de empilhamento é crucial para depurar problemas de z-index complexos.
