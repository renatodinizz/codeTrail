CSS

Display e Visibilidade

As propriedades display e visibility são cruciais para controlar como os elementos são renderizados na página e se eles são visíveis para o usuário.

Display

A propriedade display define o tipo de caixa de exibição que um elemento gera. É uma das propriedades mais importantes do CSS, pois afeta diretamente o comportamento do layout de um elemento, seus valores podem ser:

Block

Elementos com display: block ocupam toda a largura disponível e sempre começam em uma nova linha. Eles criam uma "quebra de linha" antes e depois de si mesmos. Exemplos comuns de elementos de bloco incluem <div>, <p>, <h1> a <h6>, <ul>, <li>.

Características:

  • Ocupa 100% da largura do contêiner pai por padrão.
  • Sempre começa em uma nova linha.
  • Aceita width, height, padding, margin.

Exemplo:

.minha-div {
  display: block;
  width: 200px;
  height: 100px;
  margin: 10px;
}

Inline

Elementos com display: inline não começam em uma nova linha e ocupam apenas a largura necessária para seu conteúdo. Eles são usados para elementos que fluem dentro do texto. Exemplos comuns de elementos de bloco incluem <span>, <a>, <strong>, <em>.

Características:

  • Não começa em uma nova linha.
  • Ocupa apenas a largura do seu conteúdo.
  • Não aceita width ou height (são ignorados).
  • Aceita padding horizontal, mas padding vertical e margin vertical não afetam o layout de outros elementos na linha.

Exemplo:

.meu-span {
  display: inline;
  padding: 5px;
  margin: 5px;
}

Inline-Block

Elementos com display: inline-block combinam características de inline e block. Eles fluem como elementos inline (não quebram a linha), mas aceitam propriedades de block como width, height, padding, e margin em todas as direções.

Características:

  • Não começa em uma nova linha.
  • Aceita width, height, padding, margin.
  • Útil para criar elementos que precisam de dimensões e espaçamento, mas que devem permanecer na mesma linha.

Exemplo:

.botao {
  display: inline-block;
  width: 120px;
  height: 40px;
  padding: 10px;
  margin: 5px;
}

None

Elementos com display: none são completamente removidos do fluxo do documento. Eles não ocupam nenhum espaço e não são renderizados na página. É como se o elemento nunca tivesse existido.

Características:

  • Oculta o elemento e remove-o do fluxo do documento.
  • Não ocupa espaço.
  • Não é acessível por leitores de tela.

Exemplo:

.elemento-oculto {
  display: none;
}

Flex

Elementos com display: flex se tornam um contêiner flexível, e seus filhos diretos se tornam itens flexíveis. Isso ativa o Flexbox, um módulo de layout unidimensional que permite distribuir e alinhar itens em um contêiner de forma eficiente. Será abordado em detalhes em uma seção posterior.

Exemplo:

.container-flex {
  display: flex;
  justify-content: center;
  align-items: center;
}

Grid

Elementos com display: grid se tornam um contêiner de grid, e seus filhos diretos podem ser posicionados em uma grade bidimensional. Isso ativa o CSS Grid Layout, um módulo de layout poderoso para criar layouts complexos. Será abordado em detalhes em uma seção posterior.

Exemplo:

.container-grid {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 10px;
}

Visibility:hidden vs Display:none

Ambas as propriedades podem ser usadas para ocultar elementos, mas o fazem de maneiras muito diferentes, com implicações significativas para o layout e a acessibilidade.

Visibility:hidden

Quando um elemento tem visibility: hidden, ele se torna invisível, mas ainda ocupa espaço no layout da página. É como se o elemento estivesse lá, mas transparente. Ele ainda afeta o posicionamento de outros elementos.

Características:

  • Oculta o elemento, mas ele continua ocupando seu espaço original no layout.
  • Não é visível, mas ainda pode ser interativo (por exemplo, eventos de clique ainda podem ser disparados se o elemento for clicável).
  • Ainda é acessível por leitores de tela.

Exemplo:

.elemento-invisivel {
  visibility: hidden;
}

display: none

Quando um elemento tem display: none, ele é completamente removido do fluxo do documento. Ele não ocupa nenhum espaço e não é renderizado na página. É como se o elemento nunca tivesse existido.

Características:

  • Oculta o elemento e o remove do fluxo do documento.
  • Não ocupa espaço.
  • Não é acessível por leitores de tela.

Exemplo:

.elemento-removido {
  display: none;
}

Comparação

Características do Visibility:hidden

  • Espaço no Layout - Sim, o elemento ainda ocupa seu espaço.
  • Renderização - Não é renderizado visualmente, mas existe.
  • Interatividade - Pode ser interativo (ex: cliques) se não houver overlay.
  • Acessibilidade - Acessível por leitores de tela.
  • Uso Comum - Ocultar elementos temporariamente sem alterar o layout

Características do Display:none

  • Espaço no Layout - Não, o elemento é removido do fluxo do documento.
  • Renderização - Não é renderizado de forma alguma.
  • Interatividade - Não é interativo.
  • Acessibilidade - Não acessível por leitores de tela.
  • Uso Comum - Remover elementos completamente do layout.

Opacity

A propriedade opacity define o nível de opacidade (transparência) de um elemento. O valor varia de 0.0 (totalmente transparente) a 1.0 (totalmente opaco). Ao contrário de visibility: hidden, opacity: 0 torna o elemento invisível, mas ele ainda ocupa espaço e é interativo.

.transparente {
  opacity: 0.5;
  /* 50% de opacidade */
}
.totalmente-invisivel {
  opacity: 0;
  /* Totalmente transparente, mas ainda ocupa espaço e é clicável */
}

Diferença entre opacity: 0 e visibility: hidden:

opacity: 0: O elemento é invisível, mas ainda é clicável e interage com eventos do mouse/toque. Ele ainda ocupa espaço no layout.

visibility: hidden: O elemento é invisível e não é clicável, mas ainda ocupa espaço no layout.

display: none: O elemento é completamente removido do layout, não é visível, não é clicável e não ocupa espaço.
